Page 2: 2 - Train

In this program, a train moves along a track. The user can start and stop the train and determine the design of the track and the train. For example, the user determines whether the track is represented by a simple curve or looks like an actual track. The user also determines how many cabooses the train has and whether the train releases smoke (or is environmentally friendly).

The user can also turn arc-length parameterization on and off. This affects the train's speed and the rail ties. Finally, the tension of the cardinal spline defining the track can be set between -1 and 1.

My favorite part of this project is the ability to control the number of cabooses. I am proud that I found a way to iterate through a list of cabooses and draw them in the correct position and orientation rather than draw each caboose individually, which would limit the number of cabooses drawn. Technically, my code could work with any number of cabooses and the limited range of the slider is arbitrary.

This applies the following:

  • Many objects moving at once:   the different sections of the train move individually
  • Non-trivial 2D Canvas animation:   the train's motion requires curves
  • Use of basic transformations:   the train is translated and rotated
  • A complex curve through multiple points   the train track
  • An object that moves along the complex curve and orients itself appropriately:   the train itself
  • Arc length parameterization:   constant train speed and rail ties
